# SafeHome AI - 대회 데모 시나리오 (3분)

> 대상: SW 경진대회 심사위원
> 데모 URL: https://safehome-ai.vercel.app
> 백엔드: https://safehome-ai.onrender.com
> 총 소요 시간: 3분 (180초)

---

## 오프닝 (20초)

**[화면: 랜딩 페이지]**

> "전세사기 피해자 36,449명. 평균 피해액 1.8억원.
> 만약 AI가 계약 전에 3분 만에 위험을 알려줬다면, 이 피해는 막을 수 있었습니다."

> "안녕하세요. **세이프홈 AI**입니다.
> 계약서 사진 한 장으로 25개 위험 항목을 자동 분석하고, 전세사기 위험 신호를 탐지하는 AI 서비스입니다."

**발표 팁**: 랜딩 페이지 하단의 통계 카드(1.8억 / 72% / 3분)를 가리키며 말하면 시각적 임팩트가 높다.

## 시연 1: F등급 사기 계약 탐지 (90초)

### Step 1. 랜딩 페이지 → 업로드 화면 (10초)

**[화면: 랜딩 페이지]**

> "사용자는 '계약서 분석 시작하기' 버튼 하나만 누르면 됩니다."

- "계약서 분석 시작하기" 클릭 → 업로드 화면으로 이동

### Step 2. 샘플 선택 (10초)

**[화면: 업로드 페이지 - 하단 "샘플로 체험하기" 영역]**

> "실제 서비스에서는 계약서 사진을 업로드하지만, 오늘은 실제 전세사기 패턴을 기반으로 만든 샘플로 보여드리겠습니다."

- 하단 스크롤 → 3개 샘플 버튼 중 **빨간색 "F - 전세사기"** 클릭

**발표 팁**: "이 샘플은 2023~2024년 실제 전세사기 사건에서 반복적으로 나타난 패턴을 재구성한 것입니다"라고 언급하면 신뢰도가 올라간다.

### Step 3. 분석 결과 - 등급 영역 (15초)

**[화면: 결과 페이지 상단 - 히어로 영역]**

> "결과를 보시면, **F등급**, 안전 점수 **8점**. '매우 위험'입니다."

강조 포인트:
- **F등급** 빨간색 대형 표시 → "전세사기 위험 신호가 다수 감지되었습니다"
- 안전 점수 바가 거의 비어있는 시각적 효과 가리키기
- 빨간색 권장 행동 배지: **"계약을 즉시 중단하세요"**

> "AI가 즉각적으로 '이 계약은 즉시 중단하세요'라고 알려줍니다."

### Step 4. 보증금 회수 시뮬레이션 (15초)

**[화면: 보증금 안전도 시뮬레이션 카드]**

> "보증금 2억 8천만원에 대해 경매 시 회수 가능성을 시뮬레이션합니다. 결과는 **0%**. 한 푼도 못 돌려받습니다."

강조 포인트:
- **0%** 빨간 대형 숫자 가리키기
- 위험 요인 4개 항목 빠르게 읽기:
  - "근저당 3.6억으로 보증금보다 많고 (깡통전세)"
  - "매매 당일 전세계약 (갭투자 패턴)"
  - "보증보험 가입 불가"
  - "우선변제권 포기 조항"

### Step 5. 교차 검증 결과 (15초)

**[화면: 교차 검증 결과 카드]**

> "다음은 교차 검증입니다. 계약서와 등기부등본을 대조해서 숨겨진 위험을 찾아냅니다."

강조 포인트:
- 빨간 X 3건 (danger) 하나씩 가리키기:
  - **"매매일 = 계약일"** → "소유권 이전 당일에 전세를 놓았습니다. 전형적인 갭투자입니다"
  - **"근저당 > 보증금"** → "깡통전세 확정"
  - **"보증보험 가입 불가"** → "임대인이 거부"
- 노란 경고 1건: "건축물대장은 미제출이라 추가 확인이 필요합니다"

> "이 교차 검증이 세이프홈의 핵심입니다. 계약서만 봐서는 알 수 없는 위험을, 등기부와 대조해서 잡아냅니다."

### Step 6. 위험 카드 + 수정 특약 (25초)

**[화면: 탐지된 위험 요소 카드들]**

> "총 5개 위험 요소가 탐지되었습니다. 첫 번째를 보겠습니다."

**첫 번째 카드: C-6 당일 권리 변동 (갭투자 의심)**
- SSRI 코드 배지 가리키기: **SSRI-015 (Same-day Rights Change)**
  > "이 코드는 저희가 설계한 글로벌 표준 위험 코드입니다. 한국뿐 아니라 어떤 나라에서든 같은 코드로 같은 위험을 식별할 수 있습니다."
- 계약서 원문 (빨간 배경) 가리키기:
  > "AI가 실제 계약서에서 이 문장을 찾아냈고, 왜 위험한지 쉬운 말로 설명합니다."
- "AI 분석 근거 보기" 펼치기:
  > "탐지 방식, 법적 근거, 심각도까지 투명하게 보여줍니다."
- 하단 수정 특약의 **"복사"** 버튼 클릭:
  > "수정 특약을 복사해서 중개사에게 바로 보낼 수 있습니다. 법률 용어를 몰라도 자기 보증금을 지킬 수 있습니다."

**핵심 멘트**:

> "**5개 위험 요소를 3초 만에 탐지하고, 왜 위험한지 설명하고, 어떻게 대응할지까지 알려줍니다.**"

## 시연 2: A등급 안전 계약 비교 (30초)

**[뒤로 가기 → 업로드 페이지 → "A - 안전" 샘플 클릭]**

> "반대로 안전한 계약은 어떻게 보이는지 비교해 보겠습니다."

**[화면: A등급 결과]**

강조 포인트:
- **A등급, 95점** → 초록색 화면
- 권장 행동: "계약 진행 가능합니다"
- 보증금 회수율 **98%** → 안전 요인 4개 (근저당 없음, 7년 이상 보유 등)
- 교차 검증 **전체 통과** (초록 체크 3개)
- 탐지된 위험 요소 **0건**

> "위험이 없으면 이렇게 깨끗하게 보입니다. F등급과 비교하면 한눈에 차이가 보이시죠."

**발표 팁**: F등급과 A등급 결과를 빠르게 전환하며 대비 효과를 극대화한다. 가능하면 브라우저 탭 2개를 미리 열어두고 전환한다.

| 구간 | 시간 | 누적 |
|------|------|------|
| 오프닝 | 20초 | 0:20 |

### 각 구간 핵심 강조 포인트

1. **오프닝**: 피해 숫자를 천천히 읽어라. "삼만 육천 사백 사십구명"이 "36,449명"보다 충격적이다.
2. **F등급 시연**: 0% 회수율이 나오는 순간 1~2초 멈추고 심사위원 반응을 기다려라.
3. **교차 검증**: "매매일 = 계약일" 항목을 가리킬 때 "이걸 일반인이 알 수 있을까요?"라고 질문을 던져라.
4. **SSRI 코드**: "이 코드 하나면 미국 투자자도 한국 전세 위험을 이해합니다"라고 글로벌 가치를 강조하라.
5. **A등급 비교**: F와 A를 빠르게 전환하라. 시각적 대비가 말보다 강력하다.
6. **수정 특약 복사**: 실제로 "복사" 버튼을 눌러서 "복사됨!" 텍스트가 나오는 것을 보여줘라. 실용성을 증명한다.
7. **클로징**: 마지막 문장은 외워서 카메라(심사위원)를 보며 말해라.

### 발표 톤
- 기술 설명일 때: 자신감 있고 빠르게
- 사기 피해 언급할 때: 차분하고 진지하게
- 시연할 때: "보시면", "여기를 보시면" 등 시선 유도 표현 사용

## 리스크 대비

### 1. Render Cold Start (백엔드 첫 요청 지연)

**문제**: Render 무료 플랜은 15분 비활동 시 서버가 꺼지며, 첫 요청에 30~60초 소요

**대비법**:
- 발표 **5분 전**: 브라우저에서 `https://safehome-ai.onrender.com/api/health` 를 한 번 호출하여 서버를 깨운다
- 발표 **2분 전**: Health API가 `{"status": "ok"}` 을 반환하는 것을 확인한다
- 오프닝 멘트 중: 동료가 백엔드에 한 번 더 요청을 보내둔다 (보험)

**만약 그래도 느리면**:
> "실제 서비스에서는 클라우드 상시 운영이지만, 데모 환경이라 잠시 기다려주세요."
> (이 멘트 동안 샘플 데이터를 먼저 보여주면 된다 - 샘플은 프론트엔드에 내장되어 있어 백엔드 불필요)

### 2. 네트워크 실패

**대비법**:
- **샘플 데이터는 프론트엔드에 내장**되어 있으므로 백엔드가 완전히 죽어도 샘플 시연은 가능하다
- 업로드 페이지의 "샘플로 체험하기" 버튼은 `sessionStorage`에 로컬 데이터를 저장하고 결과 페이지로 이동하므로 **오프라인에서도 동작**한다
- 최악의 경우: `localhost`로 프론트엔드를 실행해두고 전환한다

**사전 준비**:
```bash
# 발표 장소 도착 후 즉시 실행
cd frontend && npm run dev
# http://localhost:3000 에서 백업 환경 확보
```

### 3. 화면 해상도 / 모바일 뷰

- SafeHome AI는 모바일 퍼스트 디자인이므로, **브라우저 개발자 도구(F12) → 모바일 뷰(iPhone 14 Pro)**로 시연하면 UI가 더 깔끔하다
- 또는 실제 스마트폰 화면을 미러링하면 "실제 사용자 경험"을 보여줄 수 있다

### 4. 시간 초과 대비

- 가장 먼저 컷할 구간: **시연 2 (A등급 비교)** — 15초로 줄일 수 있다
- 그 다음: **Health API 시연** — 말로만 설명하고 넘어간다
- 절대 줄이지 말 것: **시연 1 (F등급)** — 이것이 핵심이다

## Q&A 예상 질문 + 답변

### Q1. "실제 계약서가 아니라 샘플 아닌가요? 진짜 계약서도 분석할 수 있나요?"

> "네, 실제 계약서도 분석할 수 있습니다. 업로드 화면에서 사진이나 PDF를 올리면 Tesseract OCR로 텍스트를 추출하고, 동일한 파이프라인으로 분석합니다. 오늘 샘플은 실제 사기 패턴을 정확히 재현한 것이고, 분석 엔진은 동일합니다. 다만 OCR 정확도에 따라 결과가 달라질 수 있어서, 텍스트가 선명한 사진을 권장하고 있습니다."

### Q2. "Rule 기반과 LLM, 둘 중 하나만 써도 되지 않나요? 왜 하이브리드인가요?"

> "Rule 엔진만 쓰면 '일체의 이의를 제기하지 않는다' 같은 교묘한 표현을 놓칩니다. 법적으로 '우선변제권 포기'인데, 키워드에 '우선변제권'이 없으니까요. 반대로 LLM만 쓰면 할루시네이션 위험이 있고, '가압류' 같은 명확한 키워드를 놓칠 수도 있습니다. Rule은 확실한 것을 빠르게, LLM은 맥락을 깊게 — 이 조합이 놓치는 것을 최소화합니다."

### Q3. "글로벌 확장이라고 했는데, 구체적으로 어떻게 되나요?"

> "백엔드에 5개의 ABC(추상 클래스) 인터페이스가 있습니다 — IRuleEngine, ILLMAnalyzer, IOCRService, ICrossValidator, ICountryPlugin. 한국 플러그인은 `plugins/kr/` 에서 이 5개를 구현합니다. 일본을 추가하려면 `plugins/jp/` 폴더를 만들고, 일본 법령에 맞는 위험 항목과 키워드를 정의하면 됩니다. 코어 파이프라인은 변경 없이 국가 플러그인만 교체하면 됩니다. SSRI 코드를 통해 한국의 SSRI-015(당일 권리 변동)와 일본의 同日権利変動을 같은 코드로 비교할 수 있습니다."

### Q4. "이걸 실제 서비스로 출시하려면 무엇이 필요한가요?"

> "세 가지가 필요합니다. 첫째, **OCR 정확도 향상** — 현재 Tesseract 기반인데, 네이버 클로바 OCR이나 Google Document AI 같은 상용 OCR로 교체하면 정확도가 크게 올라갑니다. 둘째, **법률 검증** — 25개 위험 항목과 수정 특약 문구를 부동산 법률 전문가가 검수해야 합니다. 셋째, **실사용 데이터 확보** — 실제 계약서로 테스트를 거쳐 False Positive/Negative를 줄여야 합니다. 하지만 핵심 파이프라인과 아키텍처는 이미 완성된 상태입니다."

### Q5. "LLM API 비용이 높지 않나요? 수익 모델은?"

> "현재 Claude API 비용은 계약서 1건당 약 50~100원 수준입니다. 수익 모델은 두 가지를 구상하고 있습니다. 첫째, **프리미엄 분석** — 기본 Rule 분석은 무료, LLM 심층 분석 + 교차 검증은 유료(건당 1,000~3,000원). 전세보증금 대비 극히 소액이므로 지불 의향이 높습니다. 둘째, **B2B** — 부동산 중개 플랫폼(직방, 다방 등)에 API로 제공하여 매물 안전등급 표시 기능을 넣는 것입니다. 플러그인 아키텍처 덕분에 API 통합이 용이합니다."
